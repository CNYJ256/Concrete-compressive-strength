## 二、待完成工作（三阶段）

### 第一阶段：核心实验补充（阻击"泛化性"质疑）

这是决定论文能否被录用的关键——证明结论不是因为数据集"太简单"或"有外挂"。

#### 1.1 去 UPV 消融实验 [P0]

**目标**：证明移除 UPV 这个"外挂变量"后，简单模型仍优于复杂集成。

**方法**：
1. 在墨西哥数据集上构建两个特征子集：
   - **Full (w/ UPV)**：Design_F'c, Age, Er, UPV, Ts, Fs + 4 分类变量（当前基线）
   - **No-UPV**：Design_F'c, Age, Er, Ts, Fs + 4 分类变量（移除 UPV 及 UPV 衍生特征）
   - **No-NDT**：Design_F'c, Age + 4 分类变量（移除所有 NDT：UPV, Er, Ts, Fs）
2. 对每个特征子集运行完整的 V0-V5 消融管线：
   - V0: AdaBoost 基线
   - V1: Primary + Global (32D engineered)
   - V2: Dual Space + Global
   - V3: ACDCB Full (Dual + Piecewise)
   - V4: Raw + Piecewise
   - V5: OLS Unconstrained
3. 10 折 CV，独立 Optuna 调参 (100 trials per variant)
4. 报告指标：R², RMSE, MAE, MAPE

**核心检验假设**：
- H1：移除 UPV 后，AdaBoost 基线显著下降（R² < 0.95），但"模型升级主导增益"的结论不变
- H2：No-NDT 子集上（预测难度最大），单一 XGBoost 仍优于 ACDCB Full
- H3：特征工程在 No-UPV/No-NDT 下的相对贡献与 Full 子集一致（ΔR² < 0.005）

**产出**：一张三列对比表（Full UPV / No-UPV / No-NDT）+ 分析段落。

**预计工作量**：2-3 小时（3 特征子集 × 6 变体 × 10 折 CV × 100 trials HPO）。

---

#### 1.2 非线性元学习器验证 [P1]

**目标**：实证证明即使引入非线性融合，也无法从高度相关的模型池中提取额外增益。

**方法**：
1. 在现有 OOF 预测矩阵基础上，训练三种 Meta-regressor：
   - **Linear Stacking**（当前 SLSQP，作为对照）
   - **Random Forest Meta**：`RandomForestRegressor(n_estimators=100)` 以 OOF 矩阵为输入
   - **MLP Meta**：小型 `MLPRegressor(hidden_layer_sizes=(16,8))` 以 OOF 矩阵为输入
2. 对两个数据集（UCI + 墨西哥 Full）的四组模型池：
   - H1: 原始 GBDT 池（XGB+LGB+HGB+HGB_anchor）
   - H2: +MLP（H1 + MLP_primary）
   - H3: +GPR（H1 + GPR_primary）
   - H4: +MLP+GPR（完整异质池）
3. 报告各 Meta-regressor 的融合 R²，对比最佳单模型
4. 嵌套 CV（外 10 折生成 OOF，内 5 折训练 Meta）防止过拟合

**核心检验假设**：
- H1：在 H1 池（r_min > 0.996）上，三种 Meta 的融合 R² 差异 < 0.001
- H2：在 H4 池（r_min ≈ 0.97）上，非线性 Meta 可能略优于线性，但增益 < 0.002

**产出**：一个 Meta-regressor 对比表 + 简短分析段落。

**预计工作量**：1-2 小时（OOF 矩阵已存在，仅需 Meta 层训练）。

### 第二阶段：精炼特征空间（阻击"稻草人"质疑）

审稿人认为故意引入 C 级"假设驱动"特征构成稻草人论证。需通过更公正的特征工程对比来消除疑虑。

#### 2.1 A/B 级纯净特征空间重跑 [P0]

**目标**：证明即使仅使用物理基础扎实的 Level A/B 特征，特征工程仍无增益。

**方法**：
1. 对 UCI 数据集的 32 维工程特征空间进行分类：
   - **Level A**（已建立物理模型）：W/B (Abrams), age/maturity transforms (√t, log(t))
   - **Level B**（力学/维度合理，无唯一公认模型）：W/C, SP/B, SCM ratio, paste-to-aggregate ratio
   - **Level C**（纯假设驱动，无物理模型）：Abrams index†, maturity index†, 复合交互指数, 1/(t+1)
2. 构建三个精炼特征空间：
   - **Refined-A**：8 raw + Level A 特征（约 5 个，共 ~13 维）
   - **Refined-AB**：8 raw + Level A + Level B 特征（约 10 个，共 ~18 维）
   - **Refined-All**：当前 32D（作为对照）
3. 每个精炼空间上运行消融 V1-V4：
   - V1: Primary (refined) + Global
   - V4: Raw + Piecewise（对照）
4. 固定 XGBoost 超参（从 raw 最优配置继承），仅改变特征空间 + Optuna 微调
5. 报告 R², RMSE，与 Raw 8D 基线对比

**核心检验假设**：
- H1：Refined-AB（纯物理特征，无 C 级）的 R² 仍 ≤ Raw 8D
- H2：从 Refined-A → Refined-AB → Refined-All，R² 单调递减或持平（添加更多特征不改善）

**产出**：精炼特征空间对比表 + 分析段落，直接回应"稻草人"质疑。

**预计工作量**：1-2 小时（特征分类人工判断 + 3 空间 × 4 变体 HPO）。

---

#### 2.2 数据预处理优化 [P2]

**目标**：消除审稿人指出的"除零→中位填充"数据处理伪影。

**方法**：
1. 审查当前工程特征中产生 Inf/NaN 的除零场景：
   - W/B 当 B=0（实际上 B=C+S+FA 不会为 0，但个别样本 FA 和 S 均为 0 时 B=C>0）
   - SP/W 当 W=0（W 不会为 0，但可能极小时产生极端值）
   - 1/(t+1) 无问题
2. 对确实存在问题的特征，采用物理边界约束替代中位填充：
   - 比值类特征在分母为 0 时取该特征的物理上界
   - 对数类特征在输入为 0 时取 log(ε) 而非 log(1+x) 中的 +1
3. 对比"中位填充"与"物理约束"两种预处理下的消融结果
4. 预期差异极小（< 0.0005 R²），但需诚实报告

**产出**：预处理对比的脚注或简短段落。

**预计工作量**：0.5 小时。

### 第三阶段：深度分析与讨论（提升论文立意）

通过对实验数据的进一步挖掘，回应审稿人对"安全机制"和"冗余"的看法。

#### 3.1 验证 SLSQP"安全机制"假说 [P1]

**目标**：实证证明约束优化在基模型质量参差不齐时确实起到防止性能崩塌的作用。

**方法**：
1. 构建三组"弱模型库"变体：
   - **欠拟合库**：XGB(n_estimators=5, max_depth=2), LGB(num_leaves=4), HGB(max_iter=10), HGB_anchor 同
   - **过拟合库**：XGB(max_depth=20, min_child_weight=0), LGB(num_leaves=256), HGB(max_depth=None)
   - **噪声库**：对一个正常模型的 OOF 预测加 Gaussian 噪声（σ=5 MPa）
   - **混合质量库**：2 个正常模型 + 2 个弱模型（模拟实际工程中的模型质量参差）
2. 对每组库计算：
   - OLS 无约束权重（预期出现负权重或权重 >1）
   - SLSQP 约束权重（应截断至 simplex 内）
   - 两者在测试集上的 R²/RMSE 差异
3. 与"强模型库"（当前 ACDCB）对比：
   - 强库：OLS ≈ SLSQP（权重全正，约束非绑定）
   - 弱库：SLSQP 优于 OLS（约束防止外推）
4. 报告权重差异 + 性能差异

**核心检验假设**：
- H1：弱模型库下 OLS 产生负权重（至少一个），SLSQP 将其截断为 0
- H2：弱模型库下 SLSQP 的 R²/RMSE ≥ OLS（约束防止退化但不保证改善）
- H3：混合质量库下，SLSQP 的隐式模型选择（负权重→0）提升集成鲁棒性

**产出**：Safeguard 验证对比表 + 分析段落，嵌入 Section VI-E 或 VIII。

**预计工作量**：1 小时（OOF 矩阵已存在，弱模型仅需几行代码构建，权重优化秒级）。

**目标**：在论文中增加直观的视觉证据，展示"为什么集成没用"。

**方法**：
1. 绘制两张新图：

## 四、战略建议

### 4.1 优先级判断

**P0 不能省略**：
- Phase 1.1（去 UPV）：这是审稿人最核心的关切。不做的话 W1 无法有效回应，有被拒风险。
- Phase 2.1（A/B 级特征）：直接回应 W3"稻草人"指控。好消息是 VIF 过滤实验已部分覆盖，重跑工程量不大。

**P1 强烈建议做**：
- Phase 1.2（非线性元学习器）：回应 W2。即使结果如预期（非线性也无增益），"我们测试过并排除了这个可能性"比"逻辑上不可能"更有说服力。
- Phase 3.1（SLSQP 安全机制）：**我之前的建议有误**。这个实验不是可选的——它补全了论文对 SLSQP 的核心论述逻辑链。不做的话，"安全机制"的声称缺乏实证支撑。

**P2 做比不做好**：
- Phase 2.2（预处理优化）：工作量极小，消除一个可能被追问的点。
- Phase 3.2（可视化）：提升论文可读性和说服力，不改变结论。

### 4.2 叙事策略

本轮修订的核心叙事不应是"我们在补漏洞"，而应是：

> **"审稿人的意见帮助我们更精确地界定了结论的边界条件。补充实验证实：即使移除主导物理代理(UVP)、仅保留物理合理的工程特征、引入非线性融合策略，简单模型仍构成性能上界。"**

### 4.3 Response Letter 基调

对三条 Weakness 的回复基调：
- **W1**："我们同意需要验证，补充了去 UPV 和去 NDT 的消融实验。结果证实……"
- **W2**："我们感谢这个洞察，补充了三种元学习器对比。结果证实多样性不足是信息论瓶颈而非方法论限制。"
- **W3**："我们承认原始特征空间包含物理基础薄弱的变量。补充了仅 Level A/B 特征的精炼消融。结论不变。"

### 4.4 风险提示

1. **去 UPV 实验结果可能与预期不符**：如果在 No-UPV 子集上 ACDCB 竟然显著优于简单模型，需要坦诚报告并修正结论——但这不"坏"，而是增加了论文的 nuance。
2. **非线性元学习器可能发现微小增益**：如果在 H4 池上非线性 Meta 比线性好 >0.002 R²，需承认"异质池+非线性融合"存在边际价值但不足以改变整体结论。
3. **论文长度**：新增实验可能再增加 1-2 页。考虑将 Phase 2.2 和 3.2 的部分内容移至 Supplementary Materials。
