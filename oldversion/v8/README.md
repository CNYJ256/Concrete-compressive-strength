# v8 算法详解（跨模型受控迭代版）

## 1. 版本定位

`v8` 的目标是在 `v7`（实际内部为 v5）高基线之上继续提升，并严格执行“5轮假设驱动迭代 + 受控评估 + 阻断保护”流程。

本版核心思想：

- 保留 `10-fold KFold(shuffle=True, random_state=42)` 统一评估协议；
- 前四轮进行单模型族探索（HGB / XGBoost / LightGBM / XGBoost二次强化）；
- 第五轮引入**跨特征空间加权融合**：将 v8 新候选与 `v7` 强基线锚点联合优化；
- 继续坚持 `R²` 主指标、`RMSE` 次指标的科研决策逻辑。

---

## 2. 数据与特征

### 2.1 基础输入

与 v7 一致，使用 8 个基础变量：

- `cement`
- `slag`
- `fly_ash`
- `water`
- `superplasticizer`
- `coarse_agg`
- `fine_agg`
- `age`

目标变量：`strength`。

### 2.2 双特征空间设计（v8 关键）

为提升融合上限，v8 同时维护两套特征空间：

1. **v8 特征空间（扩展机理特征）**
   - 在 v7 特征基础上新增：`binder_to_agg_ratio`、`water_to_paste_ratio`、`cement_fraction_in_binder`、`slag_fraction_in_binder`、`flyash_fraction_in_binder`、`superplasticizer_efficiency`、`maturity_index`、`agg_binder_balance`、`age_inverse`、`age_wc_interaction`。

2. **v7 特征空间（锚点特征）**
   - 保持 v7 的成熟机理特征集合，用于训练融合锚点模型（`HGB_v7_baseline`）。

> 这一设计的核心动机：让“激进探索模型”与“稳健强基线模型”在融合层协同，降低迭代漂移风险。

## 3. 5轮科研迭代执行记录

评估协议：`10-fold KFold(shuffle=True, random_state=42)`

| 轮次 | 假设 | 模型/策略 | R²_mean | RMSE_mean | 是否接纳 |
|---|---|---|---:|---:|---|
| Iter-1 | 扩大 HGB 搜索空间 + 物理增强特征可提分 | HGB + Optuna(35) | 0.945497 | 3.808984 | 否 |
| Iter-2 | XGBoost 二阶梯度 + 正则化可提升拟合 | XGBoost + Optuna(35) | 0.945934 | 3.781660 | 否 |
| Iter-3 | LightGBM 叶子优先增长可降误差 | LightGBM + Optuna(40) | 0.945652 | 3.805196 | 否 |
| Iter-4 | XGBoost 二次强化可逼近更优偏差-方差平衡 | XGBoost + Optuna(25) | 0.945215 | 3.801014 | 否 |
| Iter-5 | 跨模型融合可降低方差并突破单模型上限 | WeightedBlend（含 v7 锚点） | **0.948725** | **3.700053** | **是（v8.1）** |

### Iter-5 最优融合权重

- `HGB_v7_baseline`: **0.590218**
- `XGBoost_Iter-2`: **0.343245**
- `HGB_Iter-1`: **0.041109**
- `LightGBM_Iter-3`: **0.025428**

解释：融合最优解自动给了 v7 强基线最高权重，同时吸收了 v8 候选对误差边界的补偿能力。

## 4. 与 v7 的最终对比（核心结论）

| 指标 | v7 | v8.1 | 变化 |
|---|---:|---:|---:|
| R²_mean | 0.947965 | **0.948725** | **+0.000759** |
| RMSE_mean | 3.740782 | **3.700053** | **-0.040728** |

结论：v8 在主指标与次指标上均实现正向提升，满足“性能提升后固化新子版本”的迭代准则。

## 5. 心路历程与踩坑记录（真实科研日志）

1. **版本漂移诊断**：先确认历史目录存在“目录名与脚本版本号不一致”问题，避免错误基线比较。
2. **外部库稳定性**：初版尝试中出现过 Windows 下并行进程中断和 fortran 运行时中断现象，后通过迭代节奏与路径重构规避。
3. **失败轮次价值**：Iter-1 到 Iter-4 虽未直接超越 v7，但提供了可用于融合的“多样性基模型”。
4. **关键突破点**：将 `v7` 强基线作为融合锚点纳入 Iter-5，使融合从“平均退化”转为“双指标同时提升”。

## 7. v9 局限性与可扩展方向

### 7.1 当前局限

- 前四轮单模型探索整体未超越 v7，说明“新增特征 + 单模型”路径存在平台期；
- 目前融合权重为全局常数，尚未做到按龄期/配比分段自适应；
- 评估仍基于单次 10-fold，鲁棒性可继续用 repeated protocol 强化。

### 7.2 下一步（v9）建议

1. **Repeated K-Fold / Bootstrap CI**：报告置信区间，增强统计稳健性结论。
2. **分段融合（Age-aware Mixture）**：按龄期区间学习局部权重，提升早强/后期强度双端性能。
3. **物理约束融合**：将水胶比单调性等软约束引入融合层目标函数。
4. **可解释性闭环**：对融合前后做 SHAP + 消融，筛除伪贡献特征，减少复杂度。
